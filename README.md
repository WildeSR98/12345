# Zapret Manager

**Менеджер для автоматической установки, настройки и обновления [Zapret](https://github.com/bol-van/zapret)** — ПО для обхода DPI-блокировок.

Основан на [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube) от Flowseal, с добавлением полноценного менеджера на C#.

---

## ⚡ Быстрый старт

1. Скачайте [последний релиз](https://github.com/WildeSR98/12345/releases)
2. Распакуйте архив в любую папку
3. Запустите `autosetup.bat` **от имени администратора**
4. Менеджер автоматически проверит обновления, предложит выбрать стратегию и установит службу

> **Требования**: Windows 10/11, .NET 8 Runtime, права администратора

---

## 📋 Возможности

### Автоматическая установка
- Проверка обновлений ядра и менеджера при запуске
- Обновление hosts и списков IPSet/lists с оригинального репозитория
- Тестирование стратегий с выбором лучшей
- Установка Windows-службы `zapret` в один клик

### Главное меню (`--menu`)

| # | Пункт | Описание |
|---|-------|----------|
| **1** | Установить службу | Выбор стратегии из `strategies/`, парсинг аргументов, создание Windows-службы |
| **2** | Удалить службы | Остановка и удаление zapret, WinDivert, WinDivert14 |
| **3** | Проверить статус | Состояние всех служб |
| **4** | Игровой фильтр | Включение/выключение фильтра TCP/UDP портов для игр |
| **5** | IPSet фильтр | Переключение ipset-all.txt (загружен / any / none) |
| **6** | Автопроверка обновлений | Включение/выключение проверки обновлений при старте |
| **7** | Обновить IPSet | Скачивание ipset-service.txt из оригинального репозитория |
| **8** | Обновить Hosts | Скачивание hosts из оригинального репозитория |
| **9** | Проверить обновления | Проверка новых версий менеджера и ядра zapret |
| **10** | Диагностика | HTTP/TLS/ping проверка доступности сайтов |
| **11** | Тест стратегий | Полное тестирование всех стратегий (2 режима) |
| **12** | Экспорт отчёта | Сохранение диагностического отчёта в файл |
| **13** | TG WS Proxy | Управление Telegram WebSocket прокси |

### Тестирование стратегий

Два режима тестирования (доступны при установке и из меню [11]):

**Режим 1 — Стандартный:**
- Для каждого bat-файла запускается `winws.exe`
- Проверяется HTTP / TLS 1.2 / TLS 1.3 + ping для каждой цели
- Считается score, выбирается лучший конфиг

```
[1/19] general (ALT).bat
──────────────────────────────────────────────────────────
  Discord Main        HTTP:OK    TLS1.2:OK    TLS1.3:OK    | Ping: 42 ms
  YouTube Web         HTTP:OK    TLS1.2:OK    TLS1.3:UNSUP | Ping: 38 ms
  Google DNS 8.8.8.8  Ping: 12 ms
```

**Режим 2 — DPI тест:**
- Через curl отправляется 64KB payload на хосты из [dpi-checkers](https://github.com/hyperion-cs/dpi-checkers)
- Проверяется паттерн «TCP 16-20 KB freeze» (upload > 0, download = 0, timeout)

```
[1/19] general (ALT).bat
──────────────────────────────────────────────────────────
  [🇷🇺]Beeline       HTTP:OK    TLS1.2:OK    TLS1.3:UNSUP
  [🇷🇺]MTS           HTTP:OK    TLS1.2:BLOCK TLS1.3:OK    ⚠ 16-20KB freeze
```

### Обновления
- **Менеджер**: проверка версии через GitHub API (`WildeSR98/12345`), скачивание и замена файлов
- **Ядро (zapret core)**: проверка версии `bin/version.txt` с оригинальным репозиторием
- **Списки (lists/)**: merge с оригинальными — новые записи добавляются, пользовательские сохраняются
- **Hosts (.service/hosts)**: обновление файла hosts из оригинального репозитория
- **IPSet (.service/ipset-service.txt)**: обновление ipset с merge

## 🏗 Структура проекта

```
├── autosetup.bat          # Точка входа — запуск менеджера
├── config.json            # Конфигурация (версии, цели проверки, фичи)
├── README.md
│
├── bin/                   # Бинарники zapret (winws.exe, WinDivert, curl)
├── strategies/            # Стратегии обхода DPI (general*.bat)
├── lists/                 # Списки доменов и IP
│   ├── list-general.txt   # Основной список доменов
│   ├── list-google.txt    # Google домены
│   ├── ipset-all.txt      # IP-адреса для фильтрации
│   └── *-user.txt         # Пользовательские дополнения
├── utils/                 # Утилиты
│   ├── targets.txt        # Цели для тестирования
│   ├── manager_version.txt
│   └── test results/      # Результаты тестов (автогенерация)
│
├── publish/               # Собранный менеджер (.exe + зависимости)
│
└── src/
    └── ZapretManager/     # Исходный код менеджера (C# / .NET 8)
        ├── Program.cs             # Точка входа, меню, setup wizard
        ├── Core/                  # Конфигурация, логирование, права админа
        ├── Diagnostics/           # Тестирование (StrategyTester, DpiChecker, AccessChecker)
        ├── Lists/                 # Управление списками (download, merge, repair)
        ├── Service/               # Управление службами (install, remove, strategy parsing)
        ├── UI/                    # Консольный интерфейс (меню, спиннеры, цвета)
        └── Updates/               # Обновления (GitHub API, скачивание, бэкапы)
```

## ⚙️ Параметры запуска

| Параметр | Описание |
|----------|----------|
| _(без параметров)_ | Мастер автоматической установки |
| `--menu` | Интерактивное меню |
| `--remove` | Удаление всех служб |
| `--reinstall` | Удаление + повторная установка |
| `--test` | Тестирование стратегий |
| `--diagnostics` | Диагностика + экспорт отчёта |
| `-silent` | Тихий режим (без интерактивных промптов) |
| `-strategy "NAME"` | Указание стратегии для тихого режима |

## 📝 Конфигурация

Файл `config.json` содержит:

- **project** — версия менеджера, URL репозиториев
- **diagnostics.checkTargets** — список целей для проверки доступности
- **features** — флаги функций (verbose logging, auto-update и т.д.)
- **update** — настройки обновлений (URLs, ветки, интервалы)

dotnet build
dotnet publish -c Release -o publish --self-contained false
