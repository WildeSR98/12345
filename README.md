# Zapret Autosetup (12345)

Автоматизированный комплекс скриптов для установки, настройки и обновления **Zapret** (ПО для обхода DPI на базе маршрутизации и модификации пакетов) с функцией самообновления.

## Особенности и Функционал

- **Умное Самообновление с GitHub**: Скрипт умеет самостоятельно проверять свежие коммиты в репозитории `WildeSR98/12345`. Если найдена новая версия, он скачает архив, остановит конфликтующие службы (такие как `WinDivert` и `zapret`), обновит файлы и предложит перезапуск — всё это автоматически и без ручной замены файлов.
- **Бесшовная Установка и Обновление Zapret**: Скрипт так же проверяет обновления для самого установленного Zapret (через сервис GitHub API Release), обеспечивая вас последней версией обхода DPI.
- **Интегрированное Тестирование**: Встроенный режим тестирования конфигураций. Скрипт проверит все доступные стратегии обхода и позволит вам в 1 клик выбрать и применить лучшую работающую конфигурацию в виде службы Windows.
- **Менеджмент Списков (Авто-очистка)**: Интеллектуальное объединение и дедупликация (`merge`) файлов списков блокировок (`ipset` и `list`). Скрипт автоматически очищает неподдерживаемые адреса (например, `0.x.x.x` и `1.0.0.0/x`), сохраняя при этом ваши кастомные комментарии и IPv6 адреса.
- **Управление Службами**: Корректная остановка драйверов пакетов WinDivert (`WinDivert64.sys`) во время любых обновлений для предотвращения ошибки `Access Denied` или заблокированных файлов.

## Как установить и запустить?

1. Перейдите в папку со скачанным или склонированным репозиторием.
2. Найдите файл **`autosetup.bat`**.
3. Нажмите по нему правой кнопкой мыши и выберите **«Запуск от имени администратора»**.
4. Все дальнейшие действия (проверка прав, скачивание обновлений скрипта, запуск тестирования и запуск как службы) скрипт сделает сам в интерактивном синем окне!

> ⚠️ **Важно**: Никогда не запускайте напрямую файл `utils/autosetup.ps1` двойным кликом. Всегда используйте **`autosetup.bat`**. Этот файл-загрузчик гарантирует запуск сценария в обход политик выполнения (`ExecutionPolicy Bypass`) и с правильными правами доступа.

## Обновление скриптов

Вам больше не нужно вручную скачивать файлы с GitHub.
При каждом запуске `autosetup.bat` скрипт:

1. Запросит хэш последнего коммита из репозитория.
2. Сравнит его с локальной версией (указанной в `utils/12345_version.txt`).
3. Предложит нажать `1` для применения обновлений. После завершения скачивания скрипт закроется и попросит вас запустить `autosetup.bat` заново, чтобы продолжить работу с уже новыми фикшами и доработками.

## Структура проекта

- `autosetup.bat` — входная точка; загрузчик, обеспечивающий выполнение скриптов с правами Администратора.
- `utils/autosetup.ps1` — "Ядро" системы: вся логика по обновлению, подмене драйверов, дедупликации списков и установке служб.
- `utils/12345_version.txt` — временный файл-маркер, в котором хранится хэш скачанного commit'а с GitHub.
- `install_autostart.ps1` — дополнительный инструмент для создания задач планировщика Windows для запуска дополнительных утилит (например, TCP/WS Proxy) при каждом входе в систему.
